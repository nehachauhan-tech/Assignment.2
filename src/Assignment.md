# Gracia FRONT-END DEVELOPER Take-Home Assignment

Thank you for applying to the Front-End Developer role at Gracia Global Advisory LLP.
As the first step in our process, we're asking all applicants to complete a short assignment. This is how we get to know your design sense, your ability to translate a design into pixel-accurate code, and how you think about building real product screens. Please read through fully before you begin.

## What This Assignment Covers
The assignment has two required tasks and one optional bonus question:
1. Task 1 — Design-to-Code: pixel-perfect React code for the FinBowl screens shared via Figma link.
2. Task 2 — Original Design: design your own customer onboarding flow for a financial tool in Figma, from a blank canvas.
3. Bonus (optional) — a short app-dev question to show how you'd connect a screen to live data.

## Task 1 — Design-to-Code (Core Task)
Using the Figma file below, build pixel-perfect, production-ready React code for the screen(s) shown in the link.
**Figma file:** [https://www.figma.com/design/2iltVLU4B1ERN9uEINL5yF/Fin-Bowl---Design--Copy-?node-id=16556-207945&t=MuDh8oARRX0DzsU6-0](https://www.figma.com/design/2iltVLU4B1ERN9uEINL5yF/Fin-Bowl---Design--Copy-?node-id=16556-207945&t=MuDh8oARRX0DzsU6-0)

**What we're looking for:**
• Pixel-accurate spacing, type, and colour — measure from Figma, don't eyeball it.
• A responsive build that holds up on a smaller laptop screen, not just the exact Figma frame size.
• Sensible handling of at least one real-world state (empty / loading / error) that isn't explicitly shown in the design.
• Clean, readable code — component structure and naming a teammate could pick up without explanation.

**Stack:** this task must be built in React (function components + hooks). We're specifically assessing React proficiency here.
**Deliverable:** a GitHub repository link or a live deployed link (Vercel/Netlify/CodeSandbox all work).

## Task 2 — Original UI/UX Design (Creativity Task)
Design your own original Customer Creation / onboarding flow for a financial tool — 1–2 screens, in Figma, from a blank canvas. This is not a code task; we want to see your raw design instincts.

**The flow should include:**
• A tabbed form — each tab holds a related group of fields (e.g. Customer Details, Contact Information, Financial Details, Documents).
• All fields you'd reasonably expect when onboarding a customer for a financial product — name, contact details, PAN/ID, address, financial/account details, etc. Use your judgement on what belongs where.
• A final review step — a summary screen or table listing every field the user has filled in across all tabs, before submission.

We're looking at how you'd structure the tabs and fields, your visual hierarchy, spacing, and how you'd approach a mini component set (buttons, inputs, cards) if this were the start of a real design system.
**Deliverable:** a Figma link shared with view access enabled ('Anyone with the link can view').

## Bonus — App Dev / Full-Stack Thinking (Optional)
In 3–5 sentences (plain text, or a short code snippet if easier), explain how you'd connect the screen you built in Task 1 to a live API — specifically, how you'd handle the loading state, an error/failed-request state, and refreshing the data. This isn't mandatory, but candidates who attempt it stand out.
